### Reference equivalence (historical)

This engine was refactored from an original flat `tess_dehazing` implementation
(a `run_3d` free function, plus a Strategy-pattern `tess_dehazing_strategy`
variant). During the refactor the two were validated as producing **identical
algorithm, constants, and numeric output** — the differences were structural
only. That original code copy is **no longer bundled** in the repository.

Structural changes the refactor made relative to that original:

| Concern | This engine (`paloma.cleaning.dehazer`) | Original flat `tess_dehazing` |
|---|---|---|
| Entry point | `dehaze` ([pipeline/orchestration.py](../../../paloma/cleaning/dehazer/pipeline/orchestration.py)) | `run_3d` free function |
| Batch worker | `build_chain()` stages ([workflow/stages.py](../../../paloma/cleaning/dehazer/workflow/stages.py)) | inline batch loop in `run_3d` |
| Numeric core | `core/` subpackage (backend, patches, airlight, guided_filter, transmission, recovery) | flat `gpu.py`, `patches.py`, `estimation.py` |
| I/O + layout | `io/` subpackage (fits_io, normalize, layout) | flat `io.py`, `output_layout.py` |
| Output filename prefix | `dehazed__` | `dehazed_3d__` |

Numeric output is now pinned on real data by
[tests/test_groundtruth.py](../../tests/test_groundtruth.py), which matches the
captured `tests/data/groundtruth/expected/` FITS (generated by the original) across the
differing prefixes (`dehazed__` vs `dehazed_3d__`) to `rtol=atol=1e-9`.

### Key references

- Y. Bahat & M. Irani, *Blind Dehazing Using Internal Patch Recurrence*, IEEE
  ICCP 2016 — the dehazing prior this pipeline adapts.
- S. G. Narasimhan & S. K. Nayar, *Contrast Restoration of Weather Degraded
  Images*, IEEE TPAMI 25(6), 2003 — the haze image-formation model (Eq. 3.4).
- K. He, J. Sun & X. Tang, *Guided Image Filtering*, ECCV 2010 — transmission-map
  refinement ([05-transmission](05-transmission.md)).
- K. He, J. Sun & X. Tang, *Single Image Haze Removal Using Dark Channel Prior*,
  IEEE TPAMI 33(12), 2011 — the DCP baseline the thesis argues is unsuitable here.
- G. R. Ricker et al., *Transiting Exoplanet Survey Satellite (TESS)*, JATIS
  1(1), 2015 — the mission and FFI data product.
